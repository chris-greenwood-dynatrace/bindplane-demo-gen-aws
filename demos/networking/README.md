# Networking Demo — Network Device Fleet (NOC)

This demo shows BindPlane managing five OpenTelemetry collectors monitoring a simulated network operations center: four edge collectors (site-core, site-edge, firewalls, loadbalancers) each receive metrics, logs, and traces from a unified `device-sim` container running in their device group, which simulates DEVICE_COUNT devices per group (core routers/Cisco, WAN routers/Juniper, firewalls/Palo Alto, load balancers/F5) — every device emits all three signals over OTLP gRPC with identical resource attributes (`service.name`, `host.name`, `device.id`, `device.type`, `device.vendor`, `net.devgroup`, `net.site`) so Dynatrace correlates metrics, logs, and traces into a single entity per device without any join configuration; metrics include per-interface counters (octets, errors, discards), utilization gauges, device CPU/memory/uptime, and group-specific extras (firewall session counts and throughput, load-balancer active connections and pool member health, WAN round-trip latency and packet loss); logs are Cisco/Juniper-style structured records covering link up/down, BGP adjacency changes, OSPF state transitions, ACL denies, CRC errors, and high-CPU warnings at varied severities; traces model multi-span `network_operation` workflows (validate → reserve → push → verify, with fault-injected rollback paths keyed by `FAILURE_RATE`) so Dynatrace Distributed Traces show end-to-end change-push latency and failure rate per device group; the gateway collector receives all signals forwarded by the four edge collectors and is the sole holder of the Dynatrace destination; all five BDOT collectors enroll to BindPlane Cloud over OpAMP and receive their pipelines automatically via `bp-apply.sh`. The demo is reusable across business variants (enterprise NOC, ISP backbone, university campus, data-center fabric, retail WAN) by adjusting device type labels and cost-center attributes; `DEVICE_COUNT`, `SCRAPE_INTERVAL_S`, `FAILURE_RATE`, and `INTENSITY` in `.env.demo` let you dial volume up or down for short or long-running demos. **All telemetry is simulator-generated; no real network devices are involved.**
